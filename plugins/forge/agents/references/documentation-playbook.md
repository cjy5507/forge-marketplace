# Documentation Playbook

Use this reference when generating delivery documentation.

## README

Include:
- Project overview
- Features
- Quick start
- Prerequisites
- Installation
- Configuration
- Usage
- Project structure

## API docs

For each public endpoint include:
- Method and path
- Auth requirements
- Parameters
- Request body
- Response
- Example
- Errors

## Component docs

For each public component include:
- Props
- Basic usage
- Options / variants
- Accessibility notes

## Deployment guide

Include:
- Required environment variables
- Build commands
- Deployment steps
- Post-deploy verification
- Troubleshooting

Prefer short sentences, active voice, and working examples.
